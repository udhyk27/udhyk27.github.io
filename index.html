[Flutter] 애니메이션 SVG 적용과 앱 구조 개선하기
▼
상황 Flutter는 애니메이션이 포함된 SVG를 기본 지원하지 않아 기존 SVG 파일을 그대로 사용할 수 없었음. 또한 무거운 초기화 로직이 main에 몰려 있어 스플래시 화면이 늦게 나타나는 문제도 있었음.
원인 flutter_svg 패키지는 정적 SVG만 렌더링 가능. Firebase 초기화·Remote Config 호출 등이 화면 표시보다 먼저 실행되어 체감 속도 저하 발생.
해결 SVG 소스를 HTML에 직접 삽입 후 WebView로 렌더링하여 애니메이션 재생. 초기화 로직을 커스텀 스플래시 위젯 내부로 이동, Future.wait으로 SharedPreferences·Remote Config·애니메이션 대기(2.5s)를 병렬 처리. Api 클래스도 Static → Singleton 패턴으로 구조 개선.

[프로그래머스] 택배 상자 꺼내기
▼
접근 처음에는 2차원 배열로 지그재그 구조를 직접 구현. 배열 크기 계산과 층마다 방향 전환 처리가 번거로워 더 나은 방법을 탐색.
개선 상자 번호로 행·열 위치를 직접 계산하는 방식으로 전환. (num-1)/w로 행, (num-1)%w로 열을 구하고 홀수 행은 지그재그 보정 적용. 2차원 배열 생성 없이 메모리·가독성 모두 개선.
인사이트 다른 풀이에서 현재 열 위치에서 다음 층 같은 세로 위치까지의 거리를 수열 공식으로 바로 계산하는 방식을 발견. 불필요한 순회 없이 필요한 위치만 점프하는 가장 깔끔한 접근이라고 느꼈음.

[Flutter] 앱을 살리며 겪은 iOS 트러블슈팅
▼
상황 장기간 방치된 Flutter 프로젝트를 최신 환경으로 업데이트하는 과정에서 iOS에서만 연속적인 크래시 발생. SIGABRT → dyld 프레임워크 로드 실패 → EXC_BAD_ACCESS → 스플래시 이후 흰 화면으로 오류가 단계적으로 이어짐.
원인 Flutter 버전(3.27.4) 자체 버그, Remote Config URL 빈 문자열 미처리, AppDelegate에서 FlutterViewController가 window에 연결되지 않은 것이 최종 원인.
해결 Flutter 3.38.5로 업그레이드 후 null·isEmpty 방어 코드 추가. AppDelegate에 FlutterViewController → UIWindow → rootViewController 연결 코드를 명시적으로 작성하여 최종 해결. iOS 흰/검은 화면 문제는 네이티브 설정보다 FlutterViewController 연결 상태를 먼저 확인해야 함을 학습.
